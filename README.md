## 💡 Overview

<a href="#qLab">QLab</a> serves theatre production, live events, television production, museum installations, theme park attractions, and any professional application where precisely timed media playback and show control are required with the reliability that live, non-repeatable performance demands.

The <a href="#qLab">theatrical production use case</a> is QLab's primary market — a theatrical show's sound design, video content, and lighting control cues organized in a QLab show file that operators execute one cue at a time throughout the performance. The predictability and reliability of QLab's cue execution — that pressing Go will execute exactly the configured cue at the configured time — is the foundation of professional theatrical audio and video production. The <a href="#qLab">live event production use case</a> applies QLab to concerts, corporate events, conferences, and any live event requiring triggered audio playback, video content, or show control with precise timing and professional reliability.

The <a href="#qLab">free tier and license model</a> provides a free QLab level with limited features for evaluation and learning — professional audio, video, and control features unlock through workspace licenses. This model allows students, educators, and new users to learn QLab on the free tier before committing to professional licensing. <a href="#qLab">Figure 53 development</a> by a team with deep theatrical production expertise produces software that understands the actual needs and workflows of theatre and event production rather than software adapted from other categories.

<a href="#qLab">Network show control</a> receives OSC triggers from external systems — lighting consoles, timecode sources, automation systems — for synchronized show control where QLab is one component of a larger production system. <a href="#qLab">Script cues</a> execute AppleScript and JavaScript for custom automation — extending QLab's built-in cue types with custom logic for specialized production requirements that standard cue types don't cover.

<a href="#qLab">Show file organization and backup</a> manages the complete media and cue structure as a packaged show file — all audio, video, and configuration data bundled for transport between systems and backup with complete show integrity. <a href="#qLab">Active development from Figure 53</a> delivers regular updates with new cue types, improved performance, and expanded control capabilities responding to the evolving needs of professional production users.

## 📌 FAQ

<details>
<summary>What licensing model does QLab use?</summary>

QLab uses workspace licenses that unlock specific feature categories — audio, video, lighting, etc. Check the Figure 53 website for current licensing and pricing.

</details>

<details>
<summary>Does QLab work with external audio interfaces?</summary>

Yes. QLab routes audio through any Core Audio-compatible audio interface connected to the Mac.

</details>

<details>
<summary>Can QLab execute multiple cues simultaneously?</summary>

Yes. Group cues trigger multiple cues in parallel — allowing simultaneous audio, video, and control cue execution.

</details>

<details>
<summary>What is the show file format in QLab?</summary>

QLab show files package all cues, media, and configuration together for reliable transport between systems and backup.

</details>

<details>
<summary>Does QLab support MIDI show control?</summary>

Yes. QLab sends and receives MIDI messages for controlling and receiving triggers from MIDI-compatible production equipment.

</details>

<details>
<summary>Is there a QLab community for learning?</summary>

Yes. Figure 53 provides documentation, tutorials, and an active community forum for QLab users.

</details>
